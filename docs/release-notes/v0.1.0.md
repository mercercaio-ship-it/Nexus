## What this release is

CreativEdge v0.1.0 is the first Windows desktop validation baseline
of a local-first multi-agent chatbot. It packages 14 agents (one
orchestrator + 13 specialists) into a single hardened Electron
desktop app with a local Fastify backend, a Vite + React + TypeScript
frontend, dynamic free-port allocation, an opt-in admin console, an
Ops console with cost dashboard + crash-report local export +
explicit backup push UX, a four-step Setup wizard, and a complete
documentation set.

All canonical phases (0 through 10) are ✅ Complete. The concise
current-state roadmap lives in [`todo.md`](../../todo.md); the full
per-slice evidence trail (Windows live walkthroughs, commit hashes,
sandbox validation, privacy/security contract enforcement) lives
in [`../roadmap-history.md`](../roadmap-history.md).

- **Phase 9** (Deployment & operations) — Windows validated 2026-05-22.
- **Phase 10** (Documentation) — docs QA validated 2026-05-22.

## Highlights

- **One chat surface, 14 voices.** Type once; **🌐 Nexus** routes
  the turn to the right specialist (or convenes 2–3 specialists and
  synthesizes a single reply for multi-domain prompts).
- **Local-first.** All persistent data lives under `~/.creativedge/`
  on your machine. No telemetry. No automatic external send.
- **Hardened Electron renderer.** `contextIsolation`,
  `nodeIntegration:false`, `sandbox:true`, `webSecurity:true`. Only
  two preload surfaces: `window.ceBridge.openExternal` (HTTPS
  allow-listed to one GitHub URL) and
  `window.ceBridge.getRuntimeConfig` (port allocation metadata
  only).
- **Dynamic free-port allocation.** Inside Electron, both the
  backend and the static-asset server bind to `127.0.0.1:0` and
  read the OS-assigned port. Browser-dev mode keeps the legacy
  fixed ports.
- **First-run Setup wizard.** Four steps onboard a new user
  (Runtime → Backups → Providers → Done).
- **Ops console with six cards.** Diagnostics, Usage & cost,
  Budget & trends (SVG bar chart + local budget configuration),
  Local logs, Crash reports (prepare → review with the 17-field
  allow-list + Copy / Download — no automatic send), Backup
  (explicit second-confirmation push modal).
- **Admin console.** Live agent inspector (Phase 7-A), safe agent
  editor with diff-preview confirm modal (Phase 7-B), memory
  diff editor (Phase 7-C), routing playground.
- **Full documentation set under `docs/`.** User guide, developer
  setup, add-an-agent, customize-an-agent, troubleshooting
  (26 sections), and the Electron release runbook.

## What's inside the installer

The Windows installer (`CreativEdge-Setup-0.1.0.exe`) packages:

- Electron 30.x shell.
- Compiled backend (Fastify + better-sqlite3 + FTS5) rebuilt
  against Electron's bundled Node ABI by the `afterPack` hook
  (Phase 9-D-B2). The source `backend-api/node_modules/` is
  **never mutated** by the packaging step.
- Compiled frontend bundle served over a localhost static HTTP
  server.
- The 14-agent content tree (`agents/` + `orchestrator/`) — every
  agent's `identity.md`, `soul.md`, `personality.md`,
  `system_prompt.md`, `config.json`, and memory templates.

## Installation

1. Download `CreativEdge-Setup-0.1.0.exe` from this Releases page.
2. Run the installer (Windows SmartScreen will warn — see Security
   caveats below).
3. Launch CreativEdge from the Start menu.
4. Complete the four-step Setup wizard on first launch.

Alternatively, download a `win-unpacked` zip (if provided), extract
it, and run `CreativEdge.exe` directly without installing.

## Security caveats

This is an unsigned Windows installer — Windows SmartScreen will
warn on first run. CreativEdge has no automatic update mechanism
(no `electron-updater`, no auto-download, no auto-install); install
new versions manually from this Releases page. Crash reports stay
on disk — they are never sent automatically. The backup push
feature uses your local Git credential helper and requires explicit
second-confirmation.

Full security policy: [`SECURITY.md`](../../SECURITY.md).

---

## Known limitations

- **Windows x64 only** — no macOS / Linux desktop binaries
  (explicit non-goal).
- **No code-signing** — SmartScreen warning on first run.
- **No `electron-updater`** — manual updates only.
- **No GitHub Actions release workflow** — releases are cut
  manually by the maintainer.
- **No admin console authentication** — the admin surface assumes
  a trusted local user (intentional per §7-D).
- **Free-text `backendLogTail` is dropped** from prepared crash
  reports until a tested redaction sanitizer lands.
