### Added — Phase 9 (Deployment & operations)

- **Phase 9-A** — Reproducible setup baseline:
  `npm run setup` / `npm run setup:electron` orchestration,
  per-subpackage install scripts, post-install help.
- **Phase 9-B** — Electron wrapper foundation: dev shell that
  spawns the backend as a child process, serves the Vite-built
  frontend over a localhost static HTTP server, opens a hardened
  `BrowserWindow` (`contextIsolation`, `nodeIntegration:false`,
  `sandbox:true`, `webSecurity:true`), and ships a Windows x64 NSIS
  packaged build target. Includes the chat-recovery patches
  (lifecycle hardening + listSessions fallback + bounded retry
  polling) and the `build-win.mjs` packaging orchestrator.
- **Phase 9-C** — First-run Setup wizard: four-step onboarding
  rendered on first launch (Runtime, Backups, Providers, Done).
- **Phase 9-D-A** — Crash-log foundation + Ops console
  (Diagnostics, Usage & cost, Update info) + read-only cost
  dashboard backed by `GET /ops/usage/summary`.
- **Phase 9-D-B1** — Manual update info polish (correct GitHub
  releases URL + Check latest release button).
- **Phase 9-D-B2** — `afterPack` ABI rebuild refactor: the
  packaged-copy `better-sqlite3` is rebuilt against Electron's
  Node ABI in `electron/dist-electron/win-unpacked/resources/`
  only; source `backend-api/node_modules/` is never mutated.
- **Phase 9-D-B3** — Explicit opt-in backup push UX behind a
  second-confirmation modal (`BackupPushConfirmModal`).
- **Phase 9-D-B4** — Safe external GitHub Releases link opening via
  the new `window.ceBridge.openExternal` preload bridge with
  HTTPS-only allow-list.
- **Phase 9-D-C1** — Cost-budget alerts + cost time-series chart
  foundation (`Budget & trends` card with SVG bars +
  `localStorage`-backed monthly budget).
- **Phase 9-D-C2** — Local-only crash-report prepare/export UX
  (Prepare → review panel with the 17-field allow-list verbatim +
  privacy notice → Copy / Download / Close). No automatic send.
- **Phase 9-D-C3** — Dynamic free-port allocation: Electron picks
  free loopback ports at runtime via `net.createServer.listen(0)`;
  preload bridge exposes `window.ceBridge.getRuntimeConfig()` to
  the renderer; `VITE_API_URL` retained as a build-time
  belt-and-suspenders fallback only.
- **Phase 9-D-C4** — Final audit + Windows packaged validation
  closure for Phase 9-D-C / Phase 9-D / Phase 9 overall.

### Security & privacy posture (carried forward unchanged)

- 127.0.0.1 only; never `0.0.0.0`; never a LAN-routable interface.
- No telemetry; no automatic external send; no upload; no
  background polling; no auto-install / auto-download.
- No `electron-updater`; no signing wiring; no GitHub Actions
  release workflow (all explicit non-goals — gated on a future
  signing-certificate decision).
- Crash reports local-only; manual prepare/export only.
- Backup push opt-in with explicit second-confirmation modal.
- External release link allow-listed to a single GitHub URL.
- `afterPack` rebuild keeps source tree un-mutated.

### Known caveats

- Windows installer is **unsigned** — SmartScreen will warn on
  first run. Code-signing is deferred.
- macOS / Linux desktop binaries are explicit non-goals today
  (the packaged-build target is Windows x64 NSIS only).
- The free-text `backendLogTail` field in prepared crash reports
  is dropped until a tested-redaction sanitizer lands.
